# Сервис проверки клиентов по 115-ФЗ

## Описание проекта

Данный проект представляет собой программный прототип сервиса проверки клиентов на предмет рисков, связанных с требованиями Федерального закона №115-ФЗ.

Приложение реализует первичную оценку клиента и операции на основе риск-ориентированного подхода. Система анализирует данные клиента, параметры операции, наличие клиента во внутреннем стоп-листе, страну клиента и назначение платежа, после чего рассчитывает итоговый балл риска и присваивает уровень риска.

Проект разработан в рамках курсовой работы на тему:

**«Проектирование сервиса проверки клиентов на предмет ФЗ №115»**

---

## Основные возможности

Приложение позволяет:

- вводить данные клиента;
- проверять корректность ИНН;
- учитывать тип клиента: физическое лицо, индивидуальный предприниматель, юридическое лицо;
- выполнять автоматическую проверку клиента по внутреннему стоп-листу;
- анализировать страну клиента по справочнику стран повышенного риска;
- анализировать назначение операции по справочнику подозрительных признаков;
- учитывать отсутствие назначения операции как риск-фактор;
- рассчитывать итоговый балл риска;
- определять уровень риска: низкий, средний или высокий;
- формировать рекомендацию для специалиста внутреннего контроля;
- автоматически сохранять результаты проверок в базу данных SQLite;
- вести журнал проверок клиентов;
- выполнять поиск и фильтрацию записей журнала;
- скачивать журнал проверок в формате CSV;
- формировать PDF-отчёт по результатам проверки;
- скачивать PDF-отчёты из журнала проверок;
- удалять отдельные проверки вместе с PDF-отчётами;
- очищать журнал проверок вместе со всеми PDF-отчётами;
- просматривать справочники, используемые при расчёте риска;
- просматривать аналитику по уровням риска.

## Установка и запуск проекта

### 1. Открытие проекта

Откройте папку проекта в Visual Studio Code или другой среде разработки.

### 2. Создание виртуального окружения

В терминале выполните команду:

```bash
python -m venv .venv
```

### 3. Активация виртуального окружения

Для Windows CMD:

```bat
.venv\Scripts\activate.bat
```

Для Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

Если в PowerShell появляется ошибка политики выполнения, можно открыть терминал в режиме CMD и использовать команду для CMD.

### 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 5. Запуск приложения

```bash
streamlit run app.py
```

После запуска приложение откроется в браузере по адресу:

```text
http://localhost:8501
```

## Логика оценки риска

Система использует риск-ориентированный подход. Итоговый балл риска складывается из нескольких факторов.

| Фактор | Баллы риска |
|---|---:|
| Клиент найден во внутреннем стоп-листе | +45 |
| Сумма операции от 600 000 рублей | +25 |
| Страна клиента относится к зоне повышенного риска | +20–30 |
| Количество операций за месяц 10 или более | +20 |
| Подозрительные признаки в назначении операции | +10–25 |
| Назначение операции не указано | +5–15 |

### Шкала риска

| Итоговый балл | Уровень риска |
|---:|---|
| 0–30 | Низкий риск |
| 31–60 | Средний риск |
| 61 и выше | Высокий риск |

## Разделы приложения

### Проверка клиента

Основной раздел приложения. Пользователь вводит данные клиента и параметры операции. После нажатия кнопки проверки система рассчитывает уровень риска, выводит причины присвоения риска, формирует рекомендацию и создаёт PDF-отчёт.

### Журнал проверок

Раздел содержит историю всех выполненных проверок. Доступны поиск, фильтрация, скачивание CSV, скачивание PDF-отчёта, удаление отдельных записей и очистка журнала.

### Аналитика

Раздел отображает статистику по проверкам и график распределения клиентов по уровням риска.

### Справочники

Раздел отображает демонстрационные справочники, используемые системой:

- внутренний стоп-лист клиентов;
- страны повышенного риска;
- подозрительные признаки в назначении операции.

### О проекте

Раздел содержит описание назначения системы, основных функций и используемых технологий.

## Особенности проекта

Приложение является учебным программным прототипом. Оно не подключается к реальным государственным базам данных и не выполняет официальную проверку клиента по внешним источникам.

В рамках проекта используются демонстрационные данные, которые имитируют работу внутреннего стоп-листа и справочников риск-факторов.
